## 快照

晨露之野 Dawnfield 已从 1.0 原始目标进入 1.0+ 扩展态。M1-M16.5、M18-M22 的实现已经落在代码中，当前重点不是继续追旧规划，而是维护一个内容量明显更大的版本。

| 项 | 当前状态 |
|---|---|
| 技术栈 | Phaser 4.1 + Vite 7 + TypeScript 5.8 |
| 运行形态 | 网页游戏，键盘 + 触控，预留手柄输入 |
| 美术音频 | 零外部资源；Canvas 程序化纹理 + WebAudio 合成 |
| 内容规模 | 18 角色、8 地图、32 武器 + 32 超武、16 被动、73 敌、8 Boss |
| 构筑系统 | 24 Arcana、7 trait、16 商店强化、晨露精华、突破模式 |
| 掉落系统 | 23 一次性道具：7 通用 + 16 地图专属 |
| 成就 | 48 当前成就 + 7 legacy 成就 |
| 存档 | SaveV2，损坏备份、自愈、debounce、pagehide flush |

## 内容清单

### 角色

- 16 个常规角色：`spark`、`rosa`、`dew`、`gale`、`lumen`、`volt`、`pebble`、`fluff`、`ember`、`luna`、`conker`、`jingle`、`ivy`、`berry`、`wisp`、`toot`。
- 2 个隐藏角色：`blobby`、`nova`。未解锁时在 CharacterSelect/Codex 中不占位不显示。
- 7 个 trait：`flicker`、`sweettooth`、`fanfare`、`collector`、`resonance`、`bouncy`、`comet`。

### 地图与敌人

- 8 张地图：`meadow`、`pond`、`hills`、`grove`、`lavender`、`bramble`、`nocturne`、`summit`。
- 敌人 73 个，行为模板 19 种。
- M15 后段威胁已落地：`exploder`、`shielder`、`summoner`，以及 5 个精英词缀 `swift`、`bulwark`、`splitting`、`gravitic`、`volley`。
- M18 后 `MapMechanicSystem` 调度多个机制模块。核心机制包括 `bloomfield`、`tide`、`wind`、`sporechain`、`pollen`、`thornwall`、`nightfall`、`beacon`；旧机制在 `systems/mechanics/legacy.ts` 作为风味机制保留。

### 武器与被动

- 基础武器 32 把，全部有对应超武和 i18n 四键。
- 原 16 把仍承担角色初始武器；M22 新增 16 把作为升级池扩充。
- 被动 16 个，仍是进化条件与构筑填充的主要来源。
- M22 美术口径已经从刀斧剑、地狱火、炸弹等偏离项改回晨光、田园、纸面童话语汇。

### Arcana

- Arcana 当前为 24 张：10 张 M9 基础卡、6 张 M13 机制卡、8 张 M21 扩展卡。
- `ARCANA.maxPerRun = 3`；调试直给不受上限。
- M19 起普通宝箱不再承担 Arcana 随机层，Arcana 获取通过开局选择与专用 Arcana 宝箱链路处理。
- 机制卡解锁不新增存档字段，仍通过 `AchievementSpec.unlockArcana` 查询成就状态。

### 掉落道具

- M19 增加 `DropItemId`：7 个通用 + 每图 2 个专属，总计 23 个。
- 掉落来源包括击杀、精英、Boss、地图机制与场景物。
- 商店新增 `fortune`，通过 `POWERUP_FX.fortune` 提升掉落概率。

### 商店与经济

- 永久强化 16 项：M10 的复活、重抽、放逐、跳过，加上 M19 的 `fortune`。
- `tests/shop.test.ts` 当前锁定总金币池为 9,195。
- 商店仍支持全额重置返还；返还不污染累计金币成就链。

## 系统状态

### 稳定契约

- `content/*` 仍是纯数据层，不依赖 Phaser。
- 所有玩家可见文案都必须走 `i18n.ts` 的 `t(key)`，新增内容必须 zh/en 同步。
- 武器伤害归账通过 WeaponManager 子上下文注入，非武器来源需要显式 `ctx.dmgLog(src, dmg)`。
- 构筑随机新代码走 `ctx.rng()`，方便后续确定性种子继续收口。
- 延迟、冷却、动画时钟走 Phaser `scene.time` 或 tween，并受 `TimeController` 管理；系统内禁止裸 `setTimeout`。

### 已知文档债

- `docs/balance/dps-M12.md` 是 16 武器时期基准。M22 扩到 32 武器后，需要重新跑 bench 并新增新版平衡记录。
- `docs/UI跨平台体验审查-专项报告.md` 仍有价值，但它是专项审查报告，不是当前架构事实来源。
- 旧规划稿已归档，不应再被新任务当作“待实现计划”使用。

## 常用校验

```bash
npm run check:i18n
npx tsc --noEmit
npm test
npm run build
```

文档-only 改动通常至少跑前两项；内容、经济、存档、系统行为改动建议补跑 `npm test` 与 `npm run build`。
