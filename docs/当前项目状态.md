## 内容状态

- 角色：30 常规 + 2 隐藏；32 把基础武器均有唯一对应起手角色；隐藏角色 `blobby`、`nova` 未解锁时不占位。
- 地图：`meadow`、`pond`、`hills`、`grove`、`lavender`、`bramble`、`nocturne`、`summit`、`orchard`、`snowbell`、`mirage`、`clockwork`。
- 武器：32 把基础武器全部具备超武、图标、i18n 四键和行为类。
- 被动：16 个，兼作进化条件和构筑成长。
- 敌人：105 个，19 行为模板，5 精英词缀。
- Boss：12 个，按地图通过 `BossSpec` 配装；每个 Boss 有独占主招 + 轻辅招，旧 ring/spread/summon/dash 仅作低频辅压；实战美术由 `BossVisual` 驱动多帧/P2/施法/受击/死亡状态。
- Arcana：24 张，12 basic + 12 mechanic，全部机制化。
- 掉落：7 通用 + 每图 3 专属（共 43），其中每图至少 1 件强杀伤专属道具；踩到即触发。
- 商店：68 项永久强化，分为基础花圃、进阶工坊、地图精研三页；总金币池由测试锁定为 70,155，其中旧基础池 9,195 保持兼容。

完整 ID 表见 `docs/reference/content-catalog.md`。

## 稳定契约

- `src/content/*` 是纯数据层，不依赖 Phaser。
- 所有玩家可见文案走 `i18n.ts` 的 `t(key)`，新增键中英同步。
- 局内状态在 `RunState`，局外状态在 `Meta`，存档统一经 `core/save`。
- `GameScene` 持有 `RunSystem[]` 编排帧序，系统通过 `CombatContext` 交互。
- Arcana 和 trait 复用 `RunModifier` 钩子；衍生伤害用 `noHook` 防递归。
- 构筑随机新代码走 `ctx.rng()`。
- 延迟、冷却、动画受 `TimeController` 管理；systems 内禁止裸 `setTimeout`。
