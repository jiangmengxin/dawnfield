# 首小时引导与解锁流程规格

> 更新日期：2026-06-14  
> 事实源：`src/content/characters.ts`、`src/content/maps.ts`、`src/content/achievements.ts`、`src/content/shop.ts`、`src/core/save/schema.ts`、`src/core/MetaState.ts`、`src/scenes/*`、`src/systems/TipSystem.ts`。  
> 外部参考：只参考《Vampire Survivors》的原则级进度结构：自动攻击、升级选择、武器与被动进化、宝箱、局外金币强化、挑战解锁角色/地图/模式。不要逐项照抄具体 ID、数值或阶段顺序。参考页面：[Vampire Survivors - Gameplay](https://en.wikipedia.org/wiki/Vampire_Survivors#Gameplay)。

## 设计目标

首小时目标是让泛新手在不进入强制教程关的前提下，理解 Dawnfield 的核心循环，并带着明确的下一局目标继续玩。基准玩家不需要玩过吸血鬼幸存者类游戏，但应能理解移动、升级、装备选择和局外购买。

首小时成功标准：

- 新档仍以 `spark` + `meadow` 单角单图开局。
- 普通新手在 2-4 局内大概率通关 `meadow`，解锁 `pond`。
- 第一小时内大概率见到首次进化、首次商店购买、至少一次精英宝箱、结果页下一目标。
- 第一小时实质解锁控制在 4-6 个早期内容：`rosa`、`dew`、`gale`、`lumen`、`volt`、`pebble` 为主，`fluff` 作为经济支线。
- 局内不显示常驻任务清单；引导主要由成就、结果页、菜单锁定条件、商店推荐和少量 toast 承担。

非目标：

- 不做强制教学关，不新增主动技能、不改变“单局操作只有移动”。
- 不新增外部图片、音频、字体或运行时依赖。
- 不回收旧玩家已解锁内容；兼容策略始终是只增不收。

## 首小时玩家旅程

| 阶段 | 玩家目标 | 系统讲授 | 主要奖励 | 实施规格 |
|---|---|---|---|---|
| 0-3 分钟 | 进入 `spark` / `meadow`，开始移动避敌 | 自动攻击、经验光珠、升级三选一 | 早期等级与金币 | 新档界面只需要突出可选卡；未解锁内容用条件承接，不解释全系统 |
| 3-8 分钟 | 见到第一波压力、拾取红心/金币/宝箱 | 生存、拾取、精英宝箱 | `swarm100`、`eliteSlayer`、`survive5` 可能达成 | `TipSystem` 保留精英宝箱提示；结果页把新解锁角色放在下一局建议附近 |
| 第 1 次结算 | 去商店花钱或查看新角色 | 局外金币强化、成就解锁 | `firstBuy`、早期 powerup 等 | 商店显示早购推荐，不强制购买；推荐顺序见下文 |
| 第 2-3 局 | 尝试新角色或继续草甸 | 初始武器差异、被动配方、满级武器 | `level20`、`maxWeapon`、`firstEvolve` | 满级武器提示必须说明“配对被动 + 宝箱”下一步 |
| 第 2-4 局 | 完成首次进化并通关草甸 | 武器进化、Boss、地图链 | `volt`、`pebble`、`pond` | 结果页优先显示 `pond` 解锁与下一目标；规则卡在此后正式露出 |
| 45-60 分钟 | 尝试 `pond` 或回草甸刷金币 | 地图机制差异、长期解锁 | `fluff` 或首批商店强化 | 菜单保留清楚锁定条件，避免同时推无尽/狂暴/突破 |

首小时不要求玩家完成所有早期成就。目标是形成清楚的学习闭环：移动生存 -> 升级构筑 -> 精英宝箱 -> 进化 -> Boss 通关 -> 商店强化 -> 新地图。

## 解锁章节

解锁节奏采用主线 + 支线。地图通关链提供方向，角色、Arcana、模式和隐藏内容提供支线目标。完整主体验目标时长为 8-12 小时，隐藏和彩蛋不计入主线完成时间。

### 0-1 小时：草甸入门

主线内容：

- 初始：`spark`、`meadow`。
- 早期角色：`rosa`、`dew`、`gale`、`lumen`、`volt`、`pebble`。
- 地图：通关 `meadow` 解锁 `pond`。
- 商店：引导首次购买，推荐 `growth`、`magnet`、`power`、`vitality`、`armor`。
- 规则卡：首图通关或首次进化后再正式露出。开局不要让新玩家同时处理 Arcana、随机、倍速、突破等模式。

首小时成就职责：

| 成就 | 当前奖励 | 教学职责 |
|---|---|---|
| `survive5` | `rosa` | 教会“活够时间也有奖励” |
| `swarm100` | `dew` | 教会击杀规模目标 |
| `eliteSlayer` | `gale` | 教会精英是宝箱/解锁入口 |
| `level20` | `lumen` | 教会经验与升级成长 |
| `firstEvolve` | `volt` | 教会武器进化是构筑核心 |
| `meadowClear` | `pebble` + `pond` | 教会通关会打开新地图 |
| `coins500` | `fluff` | 经济支线，不强制首小时达成 |

### 1-4 小时：短图扩展

主线内容：

- 地图：`pond` -> `hills`。
- 目标：让玩家理解每张地图不只是换背景，而是有新的策略轴：`tide` 要求换岛，`wind` 要求顺逆风路线规划。
- 角色：逐步放出经济、范围、速度、厚血等起手差异，避免一口气把 32 角色变成噪音。
- Arcana：basic 卡可作为默认池，mechanic 卡继续由成就解锁。此阶段只需要玩家理解“规则卡会改变单局规则”，不要求三卡同辉。

建议模式露出：

- `arcana`：`meadowClear` 或 `firstEvolve` 后显示并默认开启。
- `speed2x`：通关任意 10 分钟地图后显示。
- `random`：完成首次通关后显示，但不作为推荐目标。

### 4-8 小时：中图与构筑支线

主线内容：

- 地图：`grove`、`lavender`、`bramble`、`orchard`、`snowbell`。
- 目标：引入 20 分钟地图、trait 角色、地图专属掉落、满构筑与多进化。
- 成就：`survive15`、`level30`、`kills300`、`evolve3`、`eliteHunter`、`fullHouse`、`maxPassive`。
- 角色定位：`ember`、`luna`、`conker`、`jingle`、`ivy`、`berry`、`wisp`、`toot` 等承担“角色 trait 与构筑偏好”的教学。

建议模式露出：

- `breakthrough`：完成首次进化后可展示，完成一次通关后推荐尝试。
- `berserk`：通关 `hills` 后显示，按地图已通关状态放行。

### 8-12 小时：长图、高难与长期目标

主线内容：

- 地图：`nocturne`、`summit`、`mirage`、`clockwork`。
- 目标：玩家已经理解基础构筑，此阶段主打地图机制压力、30 分钟耐力、Arcana 搭配、狂暴与无尽。
- 高阶角色：`vorty`、`lancey`、`beebee`、`frosty`、`twirl` 等由结构性挑战或长期统计解锁。
- 长期成就：`wins10`、`wins15`、`charWins10`、`hyperClear1`、`hyperClear2`、`hyperAll`、`endless1`、`endless2`、`endless3`、`endless6`。

建议模式露出：

- `endless`：每张图通关后该图开放，菜单可显示纪录。
- `hyper`：完成短图教学后显示；具体地图仍需该图通关。
- `breakthrough`：已理解进化后开放，推荐在玩家通关后尝试。

### 保密附录：隐藏角色

隐藏角色不进入主流程，不在未解锁选择页占位。

- `blobby`：由 `secretBloom` 解锁。主流程不剧透，只在实现文档中保留测试路径。
- `nova`：由 `stargazer` 解锁。主流程不剧透，不作为 8-12 小时主线验收项。

隐藏内容的 UI 原则：未解锁时不显示存在感；解锁后可用 `reveal_<id>` 的一次性揭示动效强调惊喜。

## 实施规格

本规格优先使用现有数据和存档结构：`AchievementSpec.unlockChar`、`unlockMap`、`unlockArcana`、`Meta.hasAch`、`Meta.isUnlocked`、`tipsSeen`。除非后续确实需要记录新手任务状态，否则不要新增存档字段，也不要递增 `SAVE_VERSION`。

### 结果页下一目标

结果页是最重要的软引导出口。结算后按优先级给出一个下一目标，不做多任务清单：

1. 未购买过商店强化且金币足够时，提示去商店购买早期强化。
2. 未达成 `firstEvolve` 时，提示尝试把起手武器升满并寻找配对被动。
3. 已通关 `meadow` 且 `pond` 新解锁时，提示尝试 `pond`。
4. 已通关当前地图且下一地图解锁时，提示进入下一地图。
5. 已完成短图链后，提示尝试 Arcana、突破或狂暴。

文案必须走 `i18n.ts`，建议新增通用键而不是为每个成就硬写句子。

### 菜单模式逐步露出

当前 `MapSelect` 有 6 个独立模式 chip。新手体验目标下，建议改成按状态露出或置灰：

| 模式 | 露出条件 | 放行条件 |
|---|---|---|
| 规则 | `meadowClear` 或 `firstEvolve` | 开启后走现有 Arcana 入池逻辑 |
| 随机 | 任意普通通关 | 立即可用 |
| 倍速 | 任意 10 分钟地图通关 | 立即可用 |
| 突破 | `firstEvolve` 后露出，任意通关后推荐 | 立即可用 |
| 无尽 | 至少一张地图通关后露出 | 只对已通关地图放行 |
| 狂暴 | `hillsClear` 后露出 | 只对已通关地图放行 |

若实现成本要求更低，可以先全显示但排序为：规则、倍速、随机、突破、无尽、狂暴，并对未满足条件的模式显示统一锁定说明。

### 商店早购推荐

商店不做强制路径。推荐逻辑用于标签、排序或结果页提示：

1. `growth`：提高升级频率，帮助学会构筑。
2. `magnet`：减少拾取摩擦，特别适合触控玩家。
3. `power`：提高首图通关稳定性。
4. `vitality` 或 `armor`：降低早期失败挫败。
5. `greed`：作为玩家已经能稳定通关后的经济加速。

`revive`、`reroll`、`banish`、`skip` 仍保留，但不要作为第一小时默认推荐。它们更适合玩家理解升级池后再购买。

### 进化引导

现有 `TipSystem` 已覆盖精英宝箱和武器满级。后续实施时应补强三点：

- 升级卡底部继续显示进化配方和“可进化”状态。
- 结果页在未进化时给下一目标，而不是只显示失败/金币。
- 图鉴武器页保持基础武器、超武、配对被动关系清楚可查。

首小时必见进化不意味着脚本保送。允许用轻量早期调参提高概率，例如首图前 10 分钟经验、精英宝箱节奏、起手武器升级权重、首图压力曲线，但不要加入强制发牌或固定教学局。

### 兼容策略

- 旧档已解锁角色、地图、成就、Arcana 展示权不回收。
- 降低门槛时可在启动或结算补评成就；提高门槛只影响新玩家。
- 已写入 `unlocked` 的内容不因新条件变化删除。
- 如果新增提示节流，优先复用 `tipsSeen` 字符串，不新增存档字段。

## 验收与验证

文档阶段：

```bash
npm run check:docs
```

后续实施阶段：

```bash
npm run check:docs
npm run check:i18n
npx tsc --noEmit
npm test
npm run build
```

首小时手动验收：

- fresh save 开局只看到 `spark` + `meadow` 作为可玩入口。
- 第 1 局能自然学习移动、升级、拾取、精英宝箱和结算。
- 2-4 局内普通玩家路径大概率完成首次进化并通关 `meadow`。
- 第一小时内出现首次商店购买、至少 4 个早期实质解锁、`pond` 解锁。
- 结果页始终给一个明确下一目标，不在局内塞常驻任务。
- 模式入口不会在新档第一屏同时推无尽、狂暴、随机、倍速、突破。

UI 验收分辨率：

- 402x874。
- 1440x900。
- 320x480。
- 1260x540。

验收标准：不遮挡、不溢出、可操作、文本不压字，按钮命中区不低于 `THEME.hitMin = 44`。

旧档验收：

- 已通关地图仍可进入。
- 已解锁角色仍显示。
- 已达成成就仍存在。
- 调整后的新手露出逻辑不影响 `unlockAll` 调试开关。
