# 美术方向手册

> 事实源：`src/gfx/palette.ts`、`src/gfx/textures/*`、`src/ui/theme.ts`、`src/scenes/HUD.ts`、`src/scenes/*Select.ts`。

## 总体风格

Dawnfield 的视觉目标是明亮、温暖、纸面童话。画面以暖纸白底、粉彩主体和同系深一阶描边保证可读性；战斗可以密集，但不能阴暗、血腥或金属感过强。

硬规则：

- 禁止外部图片、字体和音频资源。
- 所有美术由 Canvas 程序化纹理生成。
- 色彩只从 `src/gfx/palette.ts` 及其地图色组取。
- UI 保持纸面圆角风格，按钮、卡片、令牌和弹窗同源。
- 图标禁止 emoji，使用 Graphics 或程序化纹理。

## 色彩体系

`PAL` 是全局基础色板，包含纸底、文字、玩家、基础敌人、武器、被动、拾取物和 UI 色。地图色组包括 `POND`、`HILLS`、`GROVE`、`LAVENDER`、`BRAMBLE`、`NOCTURNE`、`SUMMIT`。

使用规则：

- 背景纸色使用 `paperCss` 或地图 `paperCss`。
- 主体色使用粉彩，描边使用同 hue 深一阶。
- 高威胁元素可以更深、更饱和，但仍需保持童话语汇。
- 终局地图允许更深夜色，但弹体、预警和 UI 必须保持高可读。

## 程序化纹理

纹理入口：

- `src/gfx/textures/core.ts`：基础绘制工具、blob、眼睛、柔光、星形、花瓣。
- `characters.ts`：角色配方与四帧动画生成。
- `mapassets.ts`：地图敌人、装饰、弹体和懒生成资源。
- `weapons.ts`：武器弹体、区域和特殊地皮。
- `icons.ts`：武器、被动、Arcana、商店等图标。
- `misc.ts`：拾取物、掉落道具、宝箱等杂项。

`ensureMapAssets(scene, mapId)` 负责地图资源懒生成；`releaseMapAssets(scene, keep)` 负责释放其他地图纹理，降低长时间运行的显存压力。

## 角色与敌人边界

角色：

- 径向渐变主体。
- 异形剪影：round/drop/gem/stone/egg 等。
- 大双高光眼、腮红、专属饰件。
- 移动拖尾粒子。
- 四帧动效：姿态 A/B × 睁眼/眨眼。

敌人：

- 扁平粉彩圆团。
- 小点眼，静态单帧。
- 用形体、调色、表情和少量饰件区分。
- 不使用角色级渐变、多帧动效或大眼高光。

新增角色不要用 `blobBody` 直接伪装；新增敌人不要做成角色质感。

## 地图视觉

每张地图要同时提供三类视觉信号：

- 纸底色和装饰层：决定环境气质。
- 敌人池调色：让地图敌人一眼属于该区域。
- 机制视觉：让玩家理解当前策略轴。

地图示例：

| 地图 | 视觉关键词 | 核心机制表现 |
|---|---|---|
| 晨光草甸 | 暖纸、青草、花朵 | 花圃成长、绽放奖励 |
| 露珠池塘 | 水蓝、荷叶、涟漪 | 涨潮与荷叶岛 |
| 晚霞山岗 | 麦浪、暖橙、风线 | 定向山风 |
| 萤暮林地 | 苔绿、蘑菇、萤光 | 孢子云连锁 |
| 紫露花田 | 薰衣草、花粉、蜂影 | 花粉带叠层 |
| 莓果灌丛 | 莓红、橄榄、刺篱 | 荆棘墙 |
| 星语夜原 | 夜蓝、星尘、月光 | 夜幕光圈与星屑 |
| 破晓之巅 | 暖金、晨光、余影 | 破晓烽台 |

## UI 与图标

UI 使用 `src/ui/theme.ts` 的统一尺寸、圆角、字号和触控命中区。常见规则：

- 触控目标不低于 44px。
- 成组按钮使用统一规格。
- 卡片用于重复项目、弹窗和工具；页面大区不要嵌套卡片。
- 技能槽位是圆形令牌，图标居中并保持等比。
- 文案较长时优先缩字号、换行或调整布局，不能溢出容器。

图标设计：

- 武器图标要表达基础形态，不必画出完整行为。
- 超武由 i18n 和图鉴描述表达，图标通常继承基础武器。
- Arcana 图标使用方形卡牌语汇，强调机制主题。
- 掉落道具图标使用彩色玻璃令牌底 + 白色 glyph。

## 视觉债与验收

当前 UI 专项报告的原文归档在 `docs/archive/2026-06-14/pre-handbook-UI跨平台体验审查-专项报告.md`。仍需关注：

- 三选一按钮高度与文案自适应。
- HUD 令牌等比缩放。
- 角色卡与地图卡的结构区分。
- 规则卡覆盖层在矮屏下的滚动和分页。
- 屏幕震动全局强度与动效克制。

UI 或美术改动必须按 `ui-ux.md` 的四个基准分辨率验收。
