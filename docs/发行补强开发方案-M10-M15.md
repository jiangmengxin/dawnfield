## 一、背景与目标

### 1.1 评审结论摘要

M9 版本工程完成度与内容基础量达到品类入门线（16 角色 / 16 武器 / 16 被动 / 8 图 / 8 Boss / 10 规则卡 / 40 成就，中英双语 100%，存档版本化迁移），但以"付费买断 + 多平台"标准衡量存在四类差距：发行配套为零（P0）、付费时长支撑不足（P1-1/6）、品类标配缺失（P1-2/3/7）、深度短板（P1-4/5/8）。当前内容量的稳妥定价仅 $2.99–3.99。

### 1.2 已拍板的发行决策

| 决策项 | 结论 |
|--------|------|
| 渠道策略 | **Steam 主发** + **Web 免费 Demo**（前 2 图引流）；移动端二阶段移植 |
| 定价目标 | **$4.99** 标准价位（对标 VS / Brotato） |
| 本期范围 | **玩法内容为主**（P1 全部 + P2 全部纳入）；P0 平台层只做选型与排期的并行规划（见第九章） |
| 新增玩法 | 无尽模式 + 难度分层（狂暴）；角色专属机制 + 机制型 Arcana |
| 无尽记录 | 仅本地最佳记录（无后端）；Steam 排行榜留待平台层接入 |
| 新手引导 | 轻量提示链路（选卡角标 + 图鉴配方 + 首局 toast），不做教程关 |
| 后续批次 | 每日挑战/固定种子、秘密内容、作曲资产、敌人剪影强化、无障碍、移动端（见第十章） |

### 2.2 跨里程碑公共契约（开工前先定，所有里程碑共享）

1. **`RunLaunchData { charId, mapId, mode: 'normal'|'endless', diff: 0|1|2 }`**——场景启动参数，M10 起预留，M11 实装。
2. **`RunResult` 扩展字段**：`mode / diff / cycle / revivesUsed / dmgTotals`（M11、M10、M15 分别消费）。
3. **`PowerUpSpec.costs?: number[]`**——显式价格表，优先于公式（M10）。
4. **存档 schema v2 一次性迁移（M11 集中落地）**：`endless`、`hyper` 两个新顶层字段 + 搭车字段 `tipsSeen`（M14 用）、`stats.winsByChar`（M13 用）。**M10 新增 PowerUpId 不需要迁移**——`sanitize()` 对 `powerUps` 未知键宽容保留（`schema.ts` 按 entries 遍历，无 id 白名单）。
5. **RunModifier 5 个新钩子 + Stats 3 个新字段（M13 落地）**：`onWeaponHit / onPlayerDamaged / modifyPlayerDamage / onCoinPicked / onEvolve`；`Stats.healMul / maxWeapons / offers`。机制卡与角色 trait 共用此地基。
6. **对齐备注**：`Stats.offers` 归 M13（ivy 四选一），与 M10 reroll 按钮共存无冲突（HUD `pickCardGeom(i, n)` 已参数化）；`surge` 是一次性 WaveEvent，**不进**无尽循环的事件重放窗口；M11 的 `endless6` 成就建议挂 `rewardCoins`，与 M13 奖励口径一致。

### 2.3 存档兼容铁律（全里程碑适用）

- **已解锁成就永不回收**：被替换的成就 id 移入 LEGACY 列表，仅已解锁者可见。
- **旧档双线回归**：每个动到存档的里程碑，验收必须包含"v1 旧档迁移 + 全新档"两条线。
- **前 2 小时体验零变化**：商店前 1,530 金币价格曲线一个数字不动；MapSelect 在无任何解锁时直接开局不弹窗。

## 三、M10 构筑操控与经济（P1-2 / P1-3 / P1-6）

> 改动面最小、不动存档版本、立即改善每一局手感；先落地金币去处，为 M11 无尽收益调参提供锚点。

### 3.1 Reroll / Banish / Skip（P1-2）

**设计意图**：品类自 VS 起的事实标准（VS/HoloCure/Brotato/20MTD 全有），承载构筑主动权 + 元进度消费点双重功能。基础次数免费送（新手即体验机制），商店永久升级追加。

**数据结构**（`src/core/RunState.ts`，开局从 `Meta.powerUpLevel` 快照）：

```ts
rerolls: number;        // = 1 + powerUpLevel('reroll')
banishes: number;       // = 1 + powerUpLevel('banish')
skips: number;          // = 1 + powerUpLevel('skip')
banished: Set<string>;  // 'w_blade' / 'p_power'，局内持久，不入存档
```

**LevelUpSystem 改动**（`src/systems/LevelUpSystem.ts`）：

- `buildOffers()` 与宝箱升级层候选生成时过滤 `banished`（放逐对宝箱升级层同样生效，保持一致性）。
- `reroll()`：次数 -1，重跑 `buildOffers()` + modifiers 链（onepath 等规则卡的 `modifyOffers` 自然兼容），重发 `hud:levelup`。
- `banish(offer)`：仅 weapon/passive 卡可放逐；次数 -1，加入 `banished`，**该卡原位重抽一张**（其余两张不动），重发事件。
- `skip()`：次数 -1，关闭面板直接回到战斗，无补偿（跳过的价值 = 留着卡池位）。
- `autoPick` 调试开关：现有早退路径（HUD 入口处）不经过新按钮，天然兼容，次数不消耗。

**UI 布局**（HUD 双形态，`src/scenes/HUD.ts`）：

- 横屏：三张大卡下方加一行两个胶囊按钮 `[↻ 重抽 ×N] [▸ 跳过 ×N]`（y = cy + ch/2 + 36）；每张卡**右上角** 28px 圆形 ✕ 角标（放逐）。次数为 0 时按钮/角标置灰。
- 竖屏：纵排条卡下方同一行两按钮（各占 45% 宽）；✕ 角标放条卡右缘垂直居中（避开 tag 文字区）。
- 键盘：`R` = 重抽，`S` = 跳过（与现有 1/2/3 选卡键并存）。
- 首次使用放逐时 `queueToast` 提示「已放逐：本局不再出现」。

**改动文件**：`core/RunState.ts`、`systems/LevelUpSystem.ts`、`scenes/HUD.ts`、`content/shop.ts`、`content/ids.ts`、`i18n.ts`。

**验收标准**：
- 三选一界面横竖屏均可重抽/放逐/跳过，次数耗尽置灰；放逐的武器整局（含宝箱升级层）不再出现。
- 商店购买后下一局次数 = 1 + 等级；autoPick 开启时一切照旧。
- onepath 规则卡的 `modifyOffers` 与重抽叠加无异常（重抽后仍只出已持有项）。

### 3.2 复活机制（P1-3）

**设计意图**：对冲 26–30 分钟长局全损挫败。本期只做商店源（VS 双轨中的被动/Arcana 源留给 M13 机制卡时复用 `run.revivesLeft` 可被任何来源 +1 的设计）。

**设计**：

- 商店条目 `revive` max2（PowerUpId 扩展，无需 schema 迁移）。
- `RunState.revivesLeft = powerUpLevel('revive')`，`revivesUsed = 0`。
- `Game.damagePlayer` 的 `hp <= 0` 分支改为：`revivesLeft > 0` → 复活演出，否则 `defeat()`。
- **复活演出**：HP 回 50% maxHp；`iframeT = 2.0`（2 秒无敌）；清屏脉冲——半径 360 内非 Boss 敌人即死（计入击杀）、Boss 击退、全部敌方弹体清除（`ProjectileSystem` 新增 `clearAll()`）；`fx.ring` 大金环 ×2 + `hitStop(0.25)` + 短和弦 SFX（victoryJingle 变体）；HUD 横幅「再次绽放！剩余复活 ×N」。
- **单局上限** = 商店等级（最高 2）。
- **无尽公平性**：记录仅本地，允许复活计入无尽记录；Result 页在无尽记录行旁标注本局 `revivesUsed`（如「♥×1」），不做记录分轨。

**改动文件**：`scenes/Game.ts`、`core/RunState.ts`、`systems/ProjectileSystem.ts`、`content/shop.ts`、`content/ids.ts`、`audio/sound.ts`（可选）、`scenes/HUD.ts`、`scenes/Result.ts`、`i18n.ts`。

**验收标准**：
- 持 1 级复活死亡：50% HP 原地复活、2 秒无敌、周围杂兵清空、弹幕清除、横幅显示；同局再死正常失败。
- 0 级时行为与现状完全一致；调试无敌开关优先（无敌时不触发复活消耗）。

### 3.3 商店经济扩展（P1-6）

**设计意图**：1,530 → 约 9,900 金币总池，元进度拉到 12–14 小时。**新手前 2 小时体验不变**——前 1,530 金币的价格曲线不动，新条目只是"看得见的远期目标"。

**价格机制**（`src/content/shop.ts`）：

```ts
interface PowerUpSpec {
  // ...现有字段
  costs?: number[];   // 显式价格表（新条目用，优先于公式）
}
// 价格函数：
// lv < 5:  base × (lv+1)         —— 现状不变
// lv >= 5: base × (lv+1) × 4     —— 高阶区间溢价 ×4
```

`powerUpSpent()` 走同一函数确定性重算，重置返还自动一致，已购等级无需迁移。

**新条目（4 项，共 3,000 金币）**：

| id | max | 价格 | 小计 | 效果 |
|----|-----|------|------|------|
| revive | 2 | 600 / 900 | 1,500 | 每局复活 +1 次 |
| reroll | 3 | 120 / 240 / 360 | 720 | 每局重抽 +1 次 |
| banish | 3 | 100 / 200 / 300 | 600 | 每局放逐 +1 次 |
| skip | 2 | 60 / 120 | 180 | 每局跳过 +1 次 |

**现有条目高阶区间（6 项 max 5→8，共 +5,376 金币）**：

| id | lv6/7/8 价格 | 小计 | 满级总效果 |
|----|--------------|------|------------|
| power | 240/280/320 | 840 | 伤害 +32% |
| vitality | 240/280/320 | 840 | HP +80 |
| haste | 288/336/384 | 1,008 | 冷却 -20% |
| area | 288/336/384 | 1,008 | 范围 +32% |
| growth | 192/224/256 | 672 | 经验 +32% |
| luck | 288/336/384 | 1,008 | 暴击 +16% |

speed / magnet / greed / armor / regen **保持原 max**——移速与经济类继续扩档会破坏平衡（greed 扩档会与无尽收益叠出通胀）。

**总池：1,530 + 3,000 + 5,376 = 9,906 金币。** 按中期收入 ~600–700/小时估算 ≈ 12–14 小时买满。调参杠杆：高阶溢价 ×4 → ×3 可压到约 8,560（实测偏肝时使用）。

**验收标准**：
- 新档前 5–8 局的购买路径（按价格升序贪心）与现版本完全一致。
- 重置返还 = 实付总额（含高阶与 costs 条目）；15 项双形态布局无溢出。
- 全买满 = 9,906；revive/reroll 等等级正确映射局内次数。

**M10 i18n 增量**：约 17 键（按钮/提示 12 + 复活 5）。

## 四、M11 无尽与狂暴（P1-1）

> schema v2 集中在本里程碑一次迁移；无尽金币衰减需在 M10 经济总池定稿后调参。

### 4.1 无尽模式（Endless）

**设计意图**：通关某图后解锁该图无尽，复用该图全部波次/事件/Boss 资产；以"5 分钟一轮"的循环节奏提供清晰记录刻度（活过第 N 轮），死亡即终局。参考 VS 无尽（Stage 循环 + 数值续涨）与 Brotato 危险等级的轮次乘区。

**核心方案：虚拟时间窗循环**（零波次数据复制）

- 设 `bossT = minutes×60`，`cycleLen = 300`（统一 5 分钟），`loopStart = bossT − cycleLen`。
- **波次**：第 k 轮（k≥1）内局部时间 `tl = (elapsed − bossT) mod cycleLen`，用虚拟时间 `vt = loopStart + tl` 查原波次表 → 循环的正是该图**最后 2–3 个峰值波次**，不含 Boss 前轻刷段。选型结论：**循环最后 N 个波次（经由时间窗），而非全表循环**——全表循环会出现"杂兵回到开局密度"的泄气段。
- **事件**：每轮开始时把 `map.events` 中 `t ∈ [loopStart, bossT]` 的事件（天然含 ring/精英/**Boss**）按轮偏移重放，`eventIdx` 每轮重置 → **Boss 每轮末重临**（同图 Boss，BossController 原样复用，不跨图借 Boss）。
- **Boss 击杀分叉**：`Game.onEnemyKilled` 中 `if (e.isBoss)` → 无尽模式不 victory，改为：宝箱掉落 + 25×衰减系数金币 + 全场磁吸脉冲 + `hud:boss` 关闭；第 ≥3 轮 Boss 带 2 名精英护卫同刷。

**难度续缩放数值初稿**（新建 `src/content/endless.ts`；现有 hpScale/dmgScale 随 elapsed 自然增长继续生效，在其上叠轮次乘区）：

```ts
ENDLESS = {
  cycleLen: 300,
  hpMulPerCycle: 0.35,        // 敌 HP ×(1 + 0.35k)
  dmgMulPerCycle: 0.18,       // 敌伤害 ×(1 + 0.18k)
  speedMulPerCycle: 0.04,     // 移速，封顶 ×1.25
  aliveMulPerCycle: 0.08,     // maxAlive，封顶 ×1.25，且全局硬上限 340
  intervalMulPerCycle: 0.05,  // 刷怪间隔 ×(1−0.05k)，地板 0.8
  bossExtraPerCycle: 0.25,    // Boss 额外 HP 乘区
  coinDecayPerCycle: 0.12,    // 金币收益 ×max(0.4, 1−0.12k) —— 防通胀
  escortFromCycle: 3, escortCount: 2,
}
```

- **XP 不衰减**：`xpForLevel` 的 ^1.85 曲线自限，升级自然变慢。
- **金币衰减**只作用于无尽分支的 `RunState.addCoins`，保护 M10 商店经济曲线。
- 性能：maxAlive 乘区结果与 340 取 min，再叠既有 `enemyCapMul × dynCapMul`，FPS 动态降档原样生效。

### 4.2 难度分层（狂暴 Hyper）

**设计意图**：每图 2 档加难（狂暴 I/II），通关本图解锁 I、通关 I 解锁 II；**与无尽完全正交**（模式 × 难度两轴独立、乘区相乘），对标 VS Hyper 与 Brotato 危险等级。

**数值初稿**（新建 `src/content/difficulty.ts`）：

| 档位 | 敌 HP | 敌伤害 | 敌移速 | 刷怪间隔 | maxAlive | 特殊 | 金币 | XP |
|------|-------|--------|--------|----------|----------|------|------|-----|
| 普通 | ×1 | ×1 | ×1 | ×1 | ×1 | — | ×1 | ×1 |
| 狂暴 I | ×1.4 | ×1.25 | ×1.08 | ×0.85 | ×1.15 | — | ×1.5 | ×1.1 |
| 狂暴 II | ×1.9 | ×1.5 | ×1.15 | ×0.7 | ×1.3 | 精英事件双刷 | ×2.0 | ×1.2 |

刻意**不加玩家限制类修正**——本作角色差异纯数值，玩家 debuff 会与低速角色叠出陷阱组合；压力全部走敌方乘区 + 精英双刷。

**解锁与组合**：
- 狂暴 I ← `Meta.hasAch(map+'Clear')`（8 图通关成就已齐备，无需新解锁字段）；狂暴 II ← `save.hyper[map] >= 1`。
- 狂暴通关同时照常触发 `<map>Clear` 链（更难覆盖更易）。
- 无尽记录单条最佳 + `diff` 注记，不做 8 图 × 3 档矩阵记录（避免 UI 与存档膨胀）。

### 4.3 存档 schema v2（一次性迁移）

```ts
// schema.ts
export const SAVE_VERSION = 2;
interface EndlessRecord { sec: number; kills: number; cycle: number; diff: number }
// SaveV2 新增：
endless: Partial<Record<MapId, EndlessRecord>>;   // 每图最佳（以 sec 判优）
hyper: Partial<Record<MapId, number>>;            // 已通关最高档 0–2
tipsSeen: string[];                               // M14 引导节流（搭车）
stats.winsByChar: Record<string, number>;         // M13 成就用（搭车）

// migrations.ts
```

**新增 5 钩子**（`src/systems/context.ts`，全部可选，零卡持有时零开销）：

```ts
/** 武器伤害结算完成后（applied>0）；连锁伤害不 re-enter（GameScene inOnHit 守卫防递归） */
onWeaponHit?(e: Enemy, applied: number, ctx: CombatContext): void;
/** 玩家实际扣血后、败北判定前（raw=护甲前，applied=实扣） */
onPlayerDamaged?(raw: number, applied: number, ctx: CombatContext): void;
/** 玩家受伤结算前改写（返回 0 = 完全免疫；闪避类机制用） */
modifyPlayerDamage?(d: number, ctx: CombatContext): number;
/** 拾取金币时 */
onCoinPicked?(value: number, ctx: CombatContext): void;
/** 武器进化时 */
onEvolve?(id: WeaponId, ctx: CombatContext): void;
```

**① 字典结构重构**（`Record<string, [zh, en]>` 元组无法扩到 10 语）：
