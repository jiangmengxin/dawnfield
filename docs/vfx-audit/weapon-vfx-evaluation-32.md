# 32 武器 VFX 与表现相似性修复复核（2026-06-14）

本文是 32 把基础武器 Lv5 与 32 个超武的 VFX 修复后复核。范围只覆盖视觉特效、表现力、相似性和移动端读感风险；不评估 DPS、经济、掉落、存档或玩家文案。

## 证据与口径

采集口径：

- 形态：基础武器 Lv5、超武。
- 场景：`staticRings`、`movingSwarm`、`singlePack`。
- 时间片：每格 0.8s / 2.0s / 3.4s 三帧，避免只看单一爆发帧。
- 视口：1440x900，`spark` + `meadow`，高 HP `blob` 标靶，临时 Playwright 驱动 live Phaser canvas。
- 采集说明：证据图已重新生成到 `docs/vfx-audit/evidence/`；未覆盖 `docs/balance/*` 数值评估图。

## 总览结论

- **P0/P1/P2/P3 开放问题：0。**
- 原 P1 共 7 项：`blade`、`spark`、`mine`、`puff`、`wand`、`breath`、`sword`，本次全部通过视觉修复关闭。
- 原 P2 小弹体、抛物 AoE、贴身环、方向近战、持续地皮相似性问题已通过本体放大、轨迹、预兆、材质化命中反馈和超武差异化关闭。
- 原 P3 健康项未大改玩法，只做低风险加固，例如 `prism` 折射节点、`rain` 纵向雨线、部分贴图轮廓强化。
- 当前仍需日后关注的不是开放缺陷：短爆发武器在固定三帧采样里仍可能错过峰值，蜂群/粒子密集武器在低端移动端需要持续看性能余量。

## 后续守则

1. 新增武器 VFX 时，每把至少保留一个非 projectile 本体的身份层：发射源、轨迹、命中印记、地皮或阵型。
2. 短爆发武器必须有前摇或残影帧，避免 0.2s 内消失。
3. 抛物 AoE 不再只用通用 `ring + p_dot`，必须体现材质差异。
4. 移动端优先复看 `swarm`、`puff`、`wisp`、`dagger`、`wand`、`frost`，它们仍是粒子密度和小体积风险最高的组。
