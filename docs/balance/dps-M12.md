# DPS 基准 — M12（2026-06-12）

> 历史基准：本文只覆盖 16 武器时期的 M12 平衡结果。M22 已扩展到 32 武器，当前平衡口径请以 `docs/balance/dps-M22-32-weapons.md` 为准。

| 项目 | 说明 |
|------|------|
| 工具 | `src/dev/bench.ts`（DEV-only；设置页调试区「DPS 基准」入口） |
| 方法 | 真实 GameScene 挂 bench 模式（无波次/机制/成就），三环 24 静止标靶（r56 / r110 / r260 各 8，血量 1e9 不死不动不退），16 武器 ×{Lv5, 进化} 各 60s 固定步长（1/60s）加速模拟 ×3 轮取均值 |
| 口径 | 玩家属性全中性（商店强化/角色偏移归一），暴击关闭（crit −0.1 抵消基础 0.1），伤害含 ±10% 浮动随机 |
| 实现要点 | 延迟伤害（delayedCall）按模拟时钟手动泵（`scene.time.preUpdate/update`）；tween 落伤害（rain/mallet）按墙钟泵 + 600ms 排空窗；隐藏页用 MessageChannel 让出主线程（setTimeout 会被 intensive throttling 钳到分钟级） |
| 耗时 | 全 96 轮 < 90 秒（验收线 3 分钟）；连跑两次单武器波动 < 3%（mine 561.8 vs 558.6 vs 559.8 ≈ 0.6%） |

## M12 复验结论（方案 4.1 验收：puff/wisp/bugle）

- **puff 1.20x** — M8 上调已生效，达标，不动。
- **wisp 0.59x → 0.66x** — 伤害 +16%（`W_WISP.dmg`）。追踪弹在静止标靶上空耗于超杀同一目标，实战（移动怪群）有效值更高，视为达标边缘，停止调参。
- **bugle 0.40x → 0.52x** — 弹伤 +24% + 种植间隔缩短（`W_BUGLE.dmg/cd`）。单株 Lv5 偏弱、进化双株 5.8x 跃升是该武器的「投资型」设计形态，不强行拉平。

## 方法论偏差（解读本表必读）

1. **静止密集标靶系统性利好 AOE**（sling/mine/blade/lantern 全部 >1.9x）：爆炸/光环每跳吃满半径内全部标靶；实战怪群散布更稀。头部比值勿直接当 nerf 依据。
2. **链式/直线贯穿系统性偏低**：spark（0.38x）链跳依赖怪群密度、prism（0.47x）一条线只穿 3 个标靶——其实战表现显著好于本表。二者不在 M12 复验清单，未调参。
3. 三环布点是对原方案双环（r120/r260）的修订：不加 r56 近战环则 petal/lantern/blade/mine 等贴身武器测不到目标。
4. 进化列受同样偏差影响；「进化/Lv5」列内部可横向比较（同一武器同一偏差）。

## 复现

设置 → 调试区 →「DPS 基准」（仅 DEV 构建）；或控制台：
`__game.scene.getScene('title').scene.start('game', { charId: 'spark', mapId: 'meadow', bench: true })`
结果输出 `console.table` 与 `window.__benchResult`（本表格式）。
