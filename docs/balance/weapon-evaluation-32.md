## 总结

- **P0：无。** 未发现会让武器系统整体失效或文档无法复现的致命问题。
- **P1：强度结构失衡。** `flask`、`swarm`、`meteor`、`bomb`、`gravity` 在至少两个场景显著高于中位；尤其 `flask` 三场景 Lv5 分别为 4.38x / 7.05x / 3.81x。
- **P1：超武跃迁失真。** `spark` 在移动怪潮从 0.13x 跳到 17.58x Lv5 倍率，`bugle`、`axe`、`frost`、`chime`、`mallet` 也存在投资后暴涨。
- **P2：低位武器缺少补偿叙事。** `prism`、`bird`、`wisp`、`vine`、`ricochet` 在多个场景低于 0.6x，但控制、覆盖或视觉反馈不足以完全解释。
- **P2：表现相似风险集中。** 抛物/落点 AoE 组、环绕贴身组、自动小弹体组和方向近战组有机制近似，建议优先拉开瞄准、时序或伤害拓扑。
- **P2：视觉证据整体非空，但部分武器弱读。** `wisp`、`bugle`、`wand`、`bird` 的弹体/伙伴在接触表里较细小；`flask`、`bomb`、`meteor`、`gravity` 的演出清晰但容易成为画面主导。

评分：1 = 明显需改，3 = 可接受但有风险，5 = 健康。P 级是后续处理优先级，不代表本次已修。

## P 级问题

| P | 问题 | 证据 | 建议 |
|---|---|---|---|
| P1 | `flask` 全场景过强 | Lv5：4.38x / 7.05x / 3.81x | 优先压池 DPS、池持续、瓶数三者之一；避免同时保留高覆盖和高 DoT。 |
| P1 | `swarm` 怪潮与单体都过强 | Lv5：怪潮 8.74x，单体 3.15x | 增加同敌命中冷却或减少蜂数/寿命；同步观察对象数量。 |
| P1 | `meteor`、`bomb`、`sling` AoE 组稳定过高 | 三场景多数 >2.6x | 先压基础 Lv5，不优先动超武幻想。 |
| P1 | `gravity` 同时高伤和强控制 | Lv5 三场景 1.82x-3.52x，另有吸拢 | 控制武器应折价，建议压持续灼蚀或内爆。 |
| P1 | `spark`、`bugle`、`axe` 超武跃迁异常 | 怪潮进化/Lv5：17.58x / 7.08x / 6.30x | 抬基础、压超武峰值，避免“未进化像惩罚、进化像开关”。 |
| P2 | `prism` 超武回报不足 | 进化/Lv5 约 1.2x | 提高折射命中可靠性，或让折射选择新目标。 |
| P2 | `petal`、`star` 启动覆盖弱 | 怪潮 Lv5 均 0.06x | 增加首次接敌能力，或明确它们只服务贴身防线。 |
| P2 | 小弹体视觉弱读 | `wisp`、`wand`、`bugle`、`bird` 接触表中辨识度低 | 提高描边、尾迹或命中闪，不必先改伤害。 |

## 相似性检验

机制近似即标风险；视觉主题不同不抵消机制重复。

| 组 | 涉及武器 | 风险 | 区分建议 |
|---|---|---|---|
| 抛物/落点 AoE | `sling`、`flask`、`bomb`、`meteor` | 高。均为自动选点、延迟落地、范围爆/池。 | `sling` 保留黏滞控制，`flask` 专注持续池，`bomb` 专注引信集束，`meteor` 保留预警大爆并降低常态覆盖。 |
| 贴身环绕/护圈 | `petal`、`lantern`、`star` | 高。均围绕玩家被动接触。 | `petal` 做多瓣接触，`lantern` 做灼烧推开，`star` 做周期远轨突入，避免都只是近身圆。 |
| 自动小弹体 | `dagger`、`wand`、`wisp`、`bugle` | 中高。都是小体积、自动索敌/射击。 | `dagger` 绑定移动方向，`wand` 稳定单体锁定，`wisp` 缓追穿透，`bugle` 固定哨塔。 |
| 方向近战 | `blade`、`vine`、`sword`、`breath`、`mallet` | 中。都朝前方释放，但形状不同。 | 用前摇、持续、线宽、击退和二段效果拉开；不要都靠简单扇形伤害。 |
| 持续区域 | `flask`、`fireball`、`gravity`、`tornado` | 中。都可形成地面/移动持续伤害。 | `gravity` 控制折价，`tornado` 游走卷击，`fireball` 走穿透轨迹，`flask` 固定池。 |
| 天降/垂直打击 | `rain`、`bolt`、`meteor` | 中。视觉都从天落下。 | `rain` 强化持续天气，`bolt` 强化随机点杀，`meteor` 强化预警大招。 |

## 性能风险

- `swarm`、`wisp`、`puff`、`bugle`：持续小实体较多，需观察对象池、命中遍历和粒子数量。
- `flask`、`fireball`、`meteor`、`gravity`、`tornado`：区域对象多且持续时间长，需关注 `ZoneSystem.count` 与重叠 DoT 查询。
- `bolt`、`meteor`、`bomb`、`mallet`：震屏、ring、burst 高频时可能加重视觉层成本。
- 当前接触表未发现空白技能或明显遮挡，但截图时间点为 0.82s，只能证明代表帧非空，不能替代长时间性能 profile。

## 调参优先级

1. 第一批：`flask`、`swarm`、`meteor`、`bomb`、`gravity`、`spark`。
2. 第二批：`sling`、`bolt`、`bugle`、`axe`、`frost`、`prism`。
3. 第三批：`petal`、`star`、`wisp`、`bird`、`vine`、`ricochet`、`lantern`、`puff`。
4. 暂缓数值，仅增强表现或说明：`tornado`、`fireball`、`sword`、`breath`、`boomerang`、`rain`、`mine`。

## 复核说明

- 本次数据为 sweep：24s ×2，用来找结构性问题。进入实际调参前，P1/P2 项应按 `60s ×3` 复核。
- `singlePack` 是故意静止的小目标组，适合测单体与锁定，不适合直接判定 `mine` 这类踩踏触发武器。
- `movingSwarm` 故意让怪从远处收束，适合暴露启动覆盖问题；贴身环绕武器在该场景偏低，需要结合真实地图波次判断。
